# Homework 3 — Fine-tuning Vision-Language Models

**Course:** Multimodal AI (MAS.S60 / 6.S985) · Spring 2026 · MIT

## Overview

This assignment investigates whether a vision-language model (VLM) can learn to select the correct API tools from a visual rendering of API documentation cards. The key idea: convert structured API schemas into styled "code card" images and fine-tune a VLM to predict function calls from these visual inputs paired with natural language instructions.

## Setup

| Component | Details |
|-----------|---------|
| **Model** | Qwen2.5-VL-3B-Instruct |
| **Fine-tuning** | LoRA (r=4, alpha=8) on attention projections (q, k, v, o) |
| **Dataset** | 800 train / 200 test samples from ToolBench |
| **Image input** | API card renderings generated by a custom renderer ([`api_card_renderer.py`](./api_card_renderer.py)) |
| **Environment** | Google Colab, A100 GPU |

## Key Work

- **API card renderer** — Built a custom image renderer that converts JSON API schemas into dark-themed code cards showing tool names, descriptions, parameters, types, and required/optional badges. This converts structured data into a visual modality for VLM consumption.
- **Dataset construction** — Created JSONL dataset pairing rendered API card images with natural language instructions and ground-truth function call labels.
- **Prompt engineering** — Designed structured output prompts (pipe-separated API names) that improved consistency and parseability of model outputs.
- **LoRA fine-tuning** — Trained with cosine LR schedule, tested learning rates (1e-4, 1e-5), batch sizes, epoch counts (up to 30), and image resolutions (up to 800px shortest edge).
- **Evaluation** — Compared pre-trained vs. fine-tuned model outputs on held-out test images.

## Key Findings

**The pre-trained model outperformed the fine-tuned model** — a surprising but instructive result:

- The fine-tuned model consistently **overfit and ignored visual content**, relying instead on text pattern memorization. It fabricated answers rather than reasoning from the API card images.
- Root causes identified:
  1. **Insufficient data**: 800 training samples is too small for stable VLM adaptation.
  2. **Visual signal degradation**: After resizing, API card images lose fine-grained text details that the model needs to read parameter names and descriptions.
- **Takeaway**: For small datasets with information-dense visual inputs, a well-prompted pre-trained VLM is preferable to a fine-tuned one. Scaling up the dataset and preserving image resolution are the most promising next steps.
